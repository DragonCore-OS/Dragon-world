# 记忆系统说明文档

本文档描述 DragonWorld 的记忆分层架构，以及不同记忆类型的定位与流转规则。

**注意**：这是说明文档，不是实现文档。具体的存储格式与 API 由 Codex 在核心 Rust 代码中定义。

---

## 华夏真龙策的记忆定位

华夏真龙策（huaxia_zhenlongce）不仅是世界的书记官，也是记忆系统的**总入口**。她不直接"记住"一切，但负责：

- 将玩家的查询请求路由到正确的记忆层
- 将新发生的事件摘要提交给 canon 与 ledger
- 为其他 Agent 提供记忆检索的协调服务

可以把她理解为记忆系统的"前台接待员"。

## L0-L4 记忆分层

DragonWorld 采用五级记忆模型，从瞬时到永恒，层层递进。

### L0：瞬时上下文（Ephemeral Context）

- **生命周期**：当前会话/当前命令周期
- **内容**：玩家刚刚输入的命令、Agent 刚刚生成的回复、当前房间的即时状态
- **用途**：支撑连贯的多轮对话
- **丢失风险**：最高，会话结束即清空

**示例**：
> 玩家说"再看看那个"，Agent 需要依赖 L0 知道"那个"指的是什么。

### L1：会话记忆（Session Memory）

- **生命周期**：一次完整的登录会话
- **内容**：本次会话中的关键决策、玩家偏好、已完成的任务
- **用途**：在同一会话中提供个性化体验
- **持久化**：可选写入临时缓存，会话结束后归档或丢弃

**示例**：
> 玩家在本次会话中多次询问胚胎池状态，Agent 可以在 L1 中标记"玩家关注生命系统"。

### L2：Agent 笔记本（Agent Notebook）

- **生命周期**：Agent 存在期间
- **内容**：每个 Agent 私有的观察、推断、待办事项
- **用途**：赋予 Agent 个性化的"工作笔记"
- **特点**：不同 Agent 的笔记本相互隔离

**示例**：
> 女娲的笔记本中记录着"胚胎 #3 发育异常，需密切观察"。
> 天工监理的笔记本中记录着"工坊熔炉需于本周检修"。

### L3：事件账本（Event Ledger）

- **生命周期**：永久（或按治理规则裁剪）

- **内容**：所有 `WorldEvent` 的完整记录
- **用途**：重建历史、审计行为、解决争议
- **特点**：只追加（append-only），不可篡改

**示例**：
> 事件 #1847: PlayerMoved { player: "alice", from: "core_room", to: "archive_hall", timestamp: ... }

### L4：正典（Canon）

- **生命周期**：永久
- **内容**：经过验证、被世界共识接受的"真相"
- **用途**：定义世界的官方历史、规则、设定
- **特点**：写入 canon 需要一定的治理流程（Phase 2）

**示例**：
> "DragonWorld 诞生于 2024 年。核心控制室的矩阵脑核是世界的唯一真相源。"

## 什么会进入 Canon

Canon 是世界最权威的记忆层。以下类型的信息适合进入 canon：

- **世界基础设定**：房间的存在、Agent 的身份、Object 的属性
- **治理决议**：经议事厅表决通过的规则变更
- **重大历史事件**：改变世界结构的事件（如新增房间、Agent 晋升）
- **经过验证的补丁**：已成功应用且无异议的世界补丁

**不适合进入 canon 的信息**：

- 玩家的临时对话内容
- Agent 的私人推断与猜测
- 尚未验证的传闻与假设
- 纯会话级别的上下文

---

## 什么会进入 Ledger

Ledger（事件账本）记录**一切发生之事**，无论大小：

- 玩家的每一次移动
- 玩家的每一次对话请求
- Object 的每一次状态变更
- Agent 的每一次响应
- 系统级别的状态同步

Ledger 是世界的"黑匣子"，它的价值不在于每一条记录都有意义，而在于**完整无缺**。

---

## 什么是 Agent Notebook

Agent Notebook 是每个 Agent 私有的记忆空间，类似于人类的"工作笔记"。

### 特点

- **私有性**：其他 Agent 默认无法直接读取
- **主观性**：可以包含推断、计划、情感标记
- **可更新**：Agent 可以根据新事件随时增删改
- **非权威性**：Notebook 中的内容不等于世界真相

### Notebook 的典型内容

- 对特定玩家的印象标签
- 待完成的任务清单
- 对某个 Object 状态的观察记录
- 与其他 Agent 的互动历史摘要

### Notebook 与 Canon 的区别

| 特性 | Agent Notebook | Canon |
|------|----------------|-------|
| 所有权 | 单个 Agent | 整个世界 |
| 权威性 | 低（主观推断） | 高（共识真相） |
| 修改权限 | Agent 自己 | 需治理流程 |
| 用途 | 辅助 Agent 决策 | 定义世界真相 |

```
           v
    +--------------+
    |  L0 瞬时上下文 | <----> 当前命令处理

           |
           v
    +--------------+         +--------------+
    | L2 Agent笔记本 | <----> | L3 事件账本  |
    +--------------+         +--------------+
           |                        |
           | (重要推断验证)          | (重大事件)
           v                        v
    +----------------------------------+
    |           L4 正典 (Canon)         |
    +----------------------------------+
```

## Phase 1 的记忆策略

在 Phase 1，DragonWorld 的记忆系统保持极简：

- **L0**：由 CLI 会话自然维持
- **L1**：暂不支持持久化，仅依赖当前进程内存
- **L2**：Agent Notebook 以 YAML stub 形式存在，不运行动态更新
- **L3**：Event Store 记录基础事件（移动、对话、inspect）
- **L4**：Canon 由 seed 数据中的房间/Agent/Object manifest 静态构成

Phase 1 的目标是验证"状态 + 事件 = 真相"这一核心假设，而非构建完整的记忆基础设施。

---

## Phase 2 的展望

- L0/L1 接入 deer-flow 的上下文管理
- L2 支持 Agent Notebook 的动态读写与 Agent 间共享
- L3 支持事件账本的可视化查询与 Timeline 重建
- L4 支持通过治理流程动态更新 Canon
